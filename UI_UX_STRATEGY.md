# UI / UX Strategy

## Product Positioning

This app should feel like a premium utility suite, not a template calculator.
The visual language is calm, warm, and practical, with one strong accent color for primary actions.

## Identity

- App name: `SimpleCalculator`
- Product description: A clean calculator and utility suite with converters, history, and a private vault.
- Personality: precise, quiet, trustworthy, and efficient.
- Tone: short labels, clear actions, no marketing fluff.

## Naming Rules

- Use direct names for screens and features.
- Prefer functional titles over decorative ones.
- Keep user-facing wording simple and specific.

Examples:

- `Calculator`
- `More Calculators`
- `Age Calculator`
- `Currency Converter`
- `Number System Conversion`
- `Private Vault`
- `History`
- `Settings`

## Screen Copy Rules

- Titles should be short.
- Supporting text should explain the value in one sentence.
- Buttons should read like actions: `Copy`, `Save`, `Convert`, `Lock`, `Open externally`.
- Avoid generic filler such as `Welcome`, `Explore`, or `Amazing features`.

## Layout Principles

- Keep a clear hierarchy: title, primary card, secondary card, then supporting actions.
- Use consistent spacing between sections.
- Use rounded cards for content surfaces.
- Keep the main action visually stronger than secondary actions.
- Make hit targets at least 44dp.

## Visual Direction

- Background: warm neutral surface.
- Cards: soft contrast, rounded corners, minimal elevation.
- Accent: orange for primary emphasis and result highlights.
- Buttons: consistent icon shape, consistent radius, no mixed styles.
- Icons: centered, balanced, and always aligned to the same grid.

## Toolbar / Header Rules

- Back button on the left.
- Screen title next to the back button.
- Optional actions on the right.
- Every header should feel like part of the same system.
- Do not use random icon sizes or different header heights per screen.

## Ad Placement Rules

- Show banners in dedicated surface areas, not directly inside content flow.
- Keep ads visible but never blocking primary interactions.
- Put native ads in discovery or hub screens, not on the most critical input path.
- Avoid inserting ads near destructive actions or file unlock flows.
- Use one clear ad placement per screen instead of many small interruptions.

## Vault / Privacy Messaging

- Present the vault as a secure, private file space.
- Avoid disguised or misleading language.
- Use calm, reassuring text around passcode and locked content.
- Be explicit when files are moved, copied, restored, or removed.

## What Makes It Stand Out

- Use exact labels instead of generic placeholders.
- Keep the orange accent reserved for important state and results.
- Make every card feel like a deliberate panel, not a default container.
- Keep motion subtle and purposeful.
- Remove anything that looks like a default Android sample screen.

## Content Example

- Main title: `Calculator`
- Subtitle: `Fast arithmetic, converters, and private tools`
- Vault description: `Store and manage private files inside the app`
- Result action: `Copy`, `Share`, `Save`
